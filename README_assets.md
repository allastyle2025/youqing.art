# 有情项目资源管理指南

本文档描述了"有情"项目的资源目录结构和管理方法，帮助您有效地组织和使用各类资源文件。

## 目录结构

目前项目已创建以下资源目录结构：

```
youqing/
├── assets/
│   └── images/
│       └── youqing_logo.png
├── .gitignore
├── README.md
├── README.rd
├── index.html
└── logo.svg
```

## 资源类型与存放位置

### 已创建的目录

- **assets/images/**：存放项目中使用的所有图片资源，包括logo、背景图、图标等。

### 未来可能需要的目录

根据项目发展需求，您可以考虑创建以下目录：

```
youqing/
├── assets/
│   ├── images/       # 图片资源
│   ├── videos/       # 视频资源
│   ├── audio/        # 音频资源（背景音乐、音效等）
│   ├── fonts/        # 自定义字体
│   ├── documents/    # 文档资源（PDF、Word等）
│   └── css/          # 自定义CSS文件（如需分离样式）
└── ...
```

## 资源引用方法

在HTML文件中引用资源时，请使用相对路径，例如：

```html
<!-- 引用图片 -->
<img src="assets/images/youqing_logo.png" alt="有情 Logo">

<!-- 引用视频（未来可能需要） -->
<video src="assets/videos/performance.mp4" controls></video>

<!-- 引用CSS（未来可能需要） -->
<link rel="stylesheet" href="assets/css/custom.css">
```

## 添加新资源的步骤

1. 将新资源文件放入对应的资源目录中
2. 在HTML或其他文件中使用相对路径引用该资源
3. 确保文件命名规范，建议使用小写字母、数字和下划线组合，避免使用空格和特殊字符

## 资源管理建议

1. **保持结构清晰**：严格按照资源类型分类存放文件
2. **优化资源大小**：图片和视频等媒体资源应进行适当压缩，以提高加载速度
3. **使用版本控制**：通过Git等版本控制工具管理资源文件的变更
4. **备份重要资源**：定期备份原始设计文件（如PSD、AI等）
5. **更新引用路径**：如果资源位置发生变化，请及时更新所有引用该资源的文件

## 注意事项

- 请确保所有添加的资源文件符合版权要求
- 对于大型资源文件（如高清视频），建议考虑使用外部存储或CDN服务
- 更新logo等核心资源后，请检查所有引用该资源的页面以确保显示正常

如果您有任何问题或需要进一步的资源管理建议，请随时联系项目负责人。
